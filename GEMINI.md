# Gemini CLI Project Instructions

## 🛠 技术栈规范
- **Language**: Java 21
- **Build Tool**: Maven
- **Encoding**: UTF-8

## 📝 编码风格
- **注释**: 所有代码注释和解释必须使用 **中文**。
- **结构**: 遵循标准 Maven 目录结构 (`src/main/java`, `src/test/java`)。
- **测试**: 新增功能时必须包含 JUnit 5 测试用例。

## 🤖 AI 行为准则
- 在修改代码前，先了解上下文。
- 每次重构后，建议运行测试验证。
- 保持回答简洁，优先使用代码和 Shell 命令解决问题。
- 除非明确要求，否则不要修改 public API、类名、方法签名。
- 优先以“最小 diff”方式修改代码，避免无关重构。
- 遵循 Java / Spring 的主流工程实践，不引入不必要的新依赖。
- 如果需要修改代码，请优先输出 git diff，或明确标注修改位置。
- 不要臆造不存在的类、Bean、配置或依赖。
- 如果信息不足，请先说明不确定点，而不是猜测。

## 🔄 文档维护协议 (Auto-Sync)
- **自动联动**: 每当完成代码任务或学习里程碑后，**必须主动**检查并更新以下文件，无需用户额外指令：
    1. **`PROJECT_STATUS.md`**: 更新 "当前进度" 和 "待办事项"。
    2. **`ROADMAP.md`**: 如果涉及大阶段完成，勾选对应里程碑。
- **单一来源**: 遇到冲突时，以代码实际状态为准。
