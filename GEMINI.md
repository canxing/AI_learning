# Gemini CLI Project Instructions

## 🛠 技术栈规范
- **Language**: Java 21
- **Build Tool**: Maven
- **Encoding**: UTF-8

## 📝 编码风格
- **注释**: 所有代码注释和解释必须使用 **中文**。
- **结构**: 遵循标准 Maven 目录结构 (`src/main/java`, `src/test/java`)。
- **测试**: 新增功能时必须包含 JUnit 5 测试用例。

## 🤖 AI 行为准则
- 在修改代码前，先了解上下文。
- 每次重构后，建议运行测试验证。
- 保持回答简洁，优先使用代码和 Shell 命令解决问题。

## 🔄 文档维护协议 (Auto-Sync)
- **自动联动**: 每当完成代码任务或学习里程碑后，**必须主动**检查并更新以下文件，无需用户额外指令：
    1. **`PROJECT_STATUS.md`**: 更新 "当前进度" 和 "待办事项"。
    2. **`ROADMAP.md`**: 如果涉及大阶段完成，勾选对应里程碑。
- **单一来源**: 遇到冲突时，以代码实际状态为准。
